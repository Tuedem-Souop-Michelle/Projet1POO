# TP3 - Gestion des réseaux et collections

## Notions étudiées

* Composition entre objets
* Collections (ArrayList)
* Ajout dynamique d’objets
* Parcours de listes (for-each)
* Relations entre classes

## Tests réalisés

* Création de plusieurs réseaux (ADMIN, TECH, WIFI)
* Ajout de plusieurs équipements (Routeur, Switch, Serveur)
* Ajout de plusieurs interfaces par équipement
* Activation des interfaces réseau
* Recherche d’équipements dans l’infrastructure
* Affichage complet de toute l’infrastructure

## Difficultés rencontrées

* Compréhension de l’utilisation des ArrayList
* Mise en relation des objets (Infrastructure → Equipements → Interfaces → Réseaux)
* Gestion des affichages multiples
* Problèmes d’encodage des caractères spéciaux

## Réponses aux questions

1. Une composition est une relation où un objet contient d’autres objets.
2. ArrayList permet de gérer dynamiquement plusieurs objets.
3. Une variable simple contient une seule valeur, une collection contient plusieurs objets.
4. Un équipement possède plusieurs interfaces pour simuler un vrai matériel réseau.
5. Une infrastructure contient plusieurs sous-réseaux pour organiser le réseau global.
6. La boucle for-each permet de parcourir facilement une collection.
7. InfrastructureReseau centralise la gestion de tous les objets du réseau.
8. Les collections sont indispensables pour gérer des systèmes complexes réels.
